# 워홀러 도움 매칭 서비스 — 기획서

---

## 1. 서비스 개요

### 문제 인식
워킹홀리데이를 떠나는 한국인들은 온라인에 정보는 많지만, 그 정보가 현지에서 실질적인 도움으로 이어지지 않는다. 기존 카톡방·페이스북 카페는 정보 공유는 되지만, 실제로 "누가 도와줄지"를 보장하지 않는다.

### 문제 인식의 배경
- 해외의 몇몇 국가 출신 커뮤니티는 네트워크가 특히 끈끈함. 한국인은 능력은 뛰어나지만 "정정당당하게 혼자 굴러가려는" 성향과 레퍼런스·추천 문화에 익숙하지 않은 경향 때문에 해외에서 손해를 보는 경우가 많음.
- 영어권 국가로 워홀을 간 경우, 영어 실력을 늘리려는 목적 때문에 같은 한국인들과 어울리는 것을 오히려 피하는 경향이 있음.
- 그러나 도움은 반드시 친한 사이에서만 오가는 것이 아니라는 관점에서, 친구가 되지 않아도 서로 돕는 것이 당연한 **느슨한 연대**를 만들고자 함.
- 대상을 한국인으로 좁힌 것은 배타성이 아니라 전략: 같은 언어·문화를 공유하는 집단에서 신뢰 형성이 더 빠르기 때문에 이 장벽이 가장 크게 느껴지는 그룹부터 시작하는 것이며, 향후 다른 국적으로 확장 가능한 구조를 지향함.

### 핵심 철학
1. **정보가 아니라 행동이 단위** — 게시글이 아니라 "요청"과 "응답"이 기본 단위.
2. **평판이 누적됨** — 도움을 준 이력이 프로필에 남아 지속적인 자산이 됨 (카톡방에서는 휘발됨).
3. **소그룹 단위로 좁혀짐** — 전체가 아니라 지역 단위 소그룹이라 심리적 거리감이 줄어듦.

## 2. 기능 구조 개요

| 구성 요소 | 역할 |
|---|---|
| 도움 크레딧 | 도움을 몇 번 요청("살")할 수 있는지 결정하는 화폐 |
| 든든 점수 | 이 사람이 얼마나 양질의 도움을 주었는지 보여주는 평가 지표. 고위험 카테고리의 접근 게이트로도 사용 |
| 도움 히스토리 | 구체적으로 어떤 도움을 주고받았는지 보여주는 이력 (판단 근거) |

- 프로필에는 크레딧·든든 점수가 바로 보이고, 도움 히스토리는 상세 정보로 들어갔을 때 노출.

## 3. 도움 카테고리 및 위험도 분류

| 유형 | 카테고리 | 크레딧 | 든든 점수 하한선 |
|---|---|---|---|
| 원격형 | 정보 제공 | 1 | 없음 |
| 원격형 | 일자리 소개(레퍼럴) | 2 | 없음 |
| 원격형 | 룸메이트·셰어하우스 연결 | 2 | 없음 |
| 원격형 | 전화 대리/동행 | 1 | 없음 |
| 물건 이전형 - 저위험 | 생활용품 나눔 | 1 | 없음 |
| 물건 이전형 - 고위험 | 집 양도 | 3 | **40점 (요청자·응답자 모두)** |
| 물건 이전형 - 고위험 | 중고차 매매 | 3 | **40점 (요청자·응답자 모두)** |
| 대면 동행형 | 이사 도움 | 2 | **40점 (요청자·응답자 모두)** |
| 대면 동행형 | 공항 픽업/마중 | 2 | **40점 (요청자·응답자 모두)** |
| 대면 동행형 | 계좌 개설 동행 | 2 | **40점 (요청자·응답자 모두)** |
| 대면 동행형 | 병원 동행 | 3 | **40점 (요청자·응답자 모두)** |

- 판단 기준은 유형 이름이 아니라 **위험도(금액 크기 또는 물리적 안전)**.
- 가입 시 든든 점수 시작값은 30점이므로, 고위험·대면 카테고리는 신입이 곧바로 이용할 수 없고 원격형·저위험 도움으로 먼저 점수를 쌓아야 함. 이는 신입의 무분별한 고위험 거래 참여를 자연스럽게 막는 안전장치로 기능.

## 4. 크레딧 시스템

- 가입(온보딩) 시 기본 크레딧 **10** 지급.
- 도움 요청 시 해당 카테고리 크레딧 차감 (크레딧 부족 시 요청 불가).
- 도움 완료 후 후기가 작성되면, 도움을 준 사람에게 해당 카테고리와 동일한 크레딧 지급.

---

## 5. 든든 점수 시스템

- 가입 시 시작값 **30점** (0~100점 척도).
- 고위험/대면 카테고리 이용 하한선 **40점** (요청자·응답자 모두 적용).
- 후기 별점에 따른 점수 변동표:

| 별점 | 점수 변동 | 의미 |
|---|---|---|
| 5점 | +1 | 양질의 도움 |
| 4점 | +1 | 양질의 도움 |
| 3점 | 0 | 보통 |
| 2점 | -3 | 불만족 |
| 1점 | -3 | 심각한 불만족 |

- 설계 의도: 양질의 도움(4~5점) 10개를 받으면 정확히 +10점(30→40). 좋은 후기 3개(+3)도 나쁜 후기 1개(-3)면 상쇄되어, 꾸준한 양질의 도움을 요구하는 구조.
- 이름 선정: '든든 점수'는 "이 사람이 있으면 마음이 놓인다"는 뉘앙스로, 서비스의 실질적 도움 지향과 맞닿아 있다고 판단하여 채택 (대안이었던 '믿음 점수'보다 선호).
- 척도 형태는 온도(당근마켓식)·퍼센티지·등급제도 검토했으나, 세밀한 차이를 표현하면서 설계가 단순한 0~100점 점수제로 결정.

## 6. 안전 장치 (대면 동행형·고위험 물건 이전형)

1. **매칭 전**: 든든 점수 하한선(40점) + 필요 시 본인인증(휴대폰 인증)
2. **매칭 시**: 상대방 도움 히스토리 열람 유도, 매칭 성사 시 자동으로 매칭 기록 저장(추후 추적 근거)
3. **진행 중**: 카테고리 진입 시 안전 수칙 안내 노출(공공장소 이용 권장, 일정 공유 권장 등). 향후 위치 공유(선택) 기능 고려 가능
4. **완료 후**: 후기에 별점과 별개로 "안전 신고" 옵션 제공. 신고 누적 시 계정 자동 정지 원칙

## 7. 법적 리스크 대응 (물건 이전형 중심)

- **원칙**: 플랫폼은 연결만 제공하고, 실제 거래(금액·물품 상태·이행 여부)는 사용자 간 책임.
- 크레딧은 실제 화폐가 아니며, 거래 금액은 플랫폼을 거치지 않고 당사자 간 직접 수수 (에스크로·결제 기능 미제공).
- 이용약관에 "플랫폼은 거래 결과에 대한 책임을 지지 않는다"는 문구 명시.
- 집 양도·중고차 매매 카테고리 진입 시 "직접 만나 확인 후 거래하세요", "공공장소에서 거래하세요" 안내 문구 노출.
- 든든 점수·후기는 "참고용 정보이며 플랫폼이 신원·물품 상태를 검증한 것이 아니다"라는 점을 명시.
- 신고 기능 운영 자체가 플랫폼의 최소한의 관리 노력을 보여주는 근거가 됨.

## 8. 페이지별 상세 기획

### 8-1. 온보딩 (계정 생성)
- 별도 로그인 없이, 온보딩 완료 자체가 계정 생성.
- 입력 항목: 국가(호주/캐나다/뉴질랜드/일본 등), 도시(텍스트), 직군(드롭다운: 농장/공장/바리스타·카페/사무직/IT/기타(직접 입력)/아직 잘 모르겠음)
- 완료 시 신규 계정 생성: 크레딧 10, 든든 점수 30으로 초기화되어 시작.
- 직군은 소그룹 그룹핑 기준이 아니라 프로필 표시용 정보.
- 온보딩은 언제든 재실행하여 새 계정을 추가로 생성 가능(다중 계정 지원).

### 8-2. 홈 화면
- 중앙에 코리 캐릭터 이미지(정적) 배치.
- 하단 메뉴 3개: 정보 게시판 / 도움 게시판 / 마이페이지.
- 우측 상단: 생성된 계정 중 선택할 수 있는 계정 전환 드롭다운.

### 8-3. 정보 게시판
- 글(제목+내용) 작성 기능.
- 글 작성 완료 시 작성자 크레딧 +1.
- 작성된 글은 리스트로 노출.

### 8-4. 도움 게시판
- 진행 중인 도움 요청 리스트 노출.
- 우측 상단 "도움 요청" 버튼 → 요청 작성 화면 이동.
- 각 요청 항목에 요청자 이름, 직군, 든든 점수 표시 + "도와줄게요!" 버튼.
- 고위험/대면 카테고리 요청은 든든 점수 40점 미만 유저에게는 요청 작성 및 응답 자체가 제한됨.

### 8-5. 도움 요청 작성 화면
- 카테고리 선택 (3절의 표 기준, 유형별로 구분되어 노출)
- 내용 텍스트 입력
- 요청 등록 시 해당 카테고리 크레딧 차감. 크레딧 부족 시 등록 불가 안내.
- 고위험/대면 카테고리 선택 시 안전 수칙 안내 문구 노출.

### 8-6. 수락 팝업 (모달)
- "도와줄게요!" 클릭 시 요청자에게 노출.
- 문구: "도움을 수락하시겠습니까?"
- 응답자 프로필 미리보기(이름, 직군, 든든 점수) 표시.
- 프로필 클릭 시 상세 정보(도움 히스토리) 화면 이동.
- 수락/거절 버튼.
- 수락 시: 게시판 노출 종료, 요청자·응답자 양쪽 도움 히스토리에 "진행중" 상태로 추가 (이 시점에는 크레딧·든든 점수 변동 없음).

### 8-7. 도움 히스토리 (마이페이지 내)
- "진행중" 상태 항목에는 "후기 작성" 버튼 노출.
- "후기 작성" 클릭 시 별점(1~5) + 코멘트 입력.
- 제출 시:
  - 해당 항목 상태 "완료"로 전환
  - 도움을 준 사람에게 카테고리 크레딧 지급
  - 도움을 준 사람의 든든 점수가 5절의 변동표에 따라 반영
  - 요청자·응답자 양쪽 히스토리에 후기 내용(별점, 코멘트) 함께 기록
- 완료 항목 상세보기 시 후기 내용 열람 가능.

### 8-8. 마이페이지
- 표시 항목: 국가, 도시, 직군, 크레딧, 든든 점수, 도움 히스토리(요청/응답 목록)
- 계정 관리 메뉴 (3개, 각각 독립적으로 동작):
  1. **계정 삭제**: 현재 선택된 계정 전체 삭제(크레딧·든든 점수·히스토리 포함). 삭제 후 다른 계정으로 자동 전환되거나, 계정이 없으면 온보딩 화면으로 이동. 다른 계정에는 영향 없음.
  2. **게시판 초기화**: 정보 게시판·도움 게시판의 모든 글 삭제. 계정의 크레딧·든든 점수·히스토리에는 영향 없음.
  3. **온보딩 화면 이동**: 데이터 변경 없이 새 계정 생성을 위한 온보딩 화면으로 이동.

## 9. 콜드 스타트(초기 밀도) 전략

- 매칭 서비스 특성상 같은 지역에 요청자·제공자가 동시에 존재해야 가치가 생기므로, 초기엔 특정 도시 하나에 집중 (Density over Reach).
- 이미 한국인 커뮤니티가 존재하지만 "실질적 도움"은 부족한 도시를 우선 선정.
- 수요자보다 공급자(정보·자원을 가진 워홀 선배)를 먼저 확보하는 것을 우선.
- 초기에는 운영자가 직접 기존 커뮤니티에서 사람을 데려와 수동으로 연결하는 것도 고려 (Do things that don't scale).
- 완전히 새로운 곳에 모으기보다, 기존 워홀 카톡방·카페에 "신뢰 레벨 붙은 확장판"으로 포지셔닝하여 얹혀 시작.
- 신입(든든 점수 0~신입 상태)도 매칭에서 배제되지 않도록 하고, 작은 기여(정보 공유 등)도 점수에 소폭 반영해 진입장벽을 낮춤.

## 10. 기존 대체재와의 차별점

- 카톡방·페이스북 카페는 익명성/휘발성이 높고, 정보가 실제 행동으로 이어지지 않으며, 평판이 축적되지 않음.
- 본 서비스는 "요청→매칭"을 기본 단위로 하고, 도움 히스토리가 프로필에 지속 축적되며, 지역 소그룹으로 좁혀 심리적 거리감을 줄임.
- 차별점은 설명보다 UI 상의 첫 경험(예: "도움 요청하기" 버튼, 매칭 시 히스토리 즉시 노출)으로 체감시키는 것이 효과적.

## 11. 수익 모델 (참고, 확정 아님)

우선순위는 수익화보다 "실질적 도움이 되는가"를 증명하는 것. 이후 고려 가능한 방향:
- 프리미엄/구독 (신뢰 레벨 연동 우선 매칭)
- 고위험 거래 수수료
- B2B 제휴 (부동산·통신사·보험사·어학원)
- 정부/기관 지원 연계
- 도움 히스토리 기반 신원 인증 서비스 (구직·렌트 등에 활용, 워홀 이후에도 지속 가능한 자산)
- 프리미엄 매칭(요청자의 긴급 요청 유료화)
- 데이터/인사이트 판매 (비식별화 전제)
- 굿즈/오프라인 밋업

가장 차별화된 방향은 서비스의 핵심 자산인 '신뢰 데이터'를 직접 자산화하는 5번(신원 인증) 또는 1번(구독) 방향으로 판단됨.

## 12. 크레딧 확장 사용처

크레딧을 "도움을 요청하는 데만" 쓸 수 있으면, 더 이상 도움이 필요 없어진 유저에게는 크레딧이 죽은 숫자가 됨. 현금화하지 않으면서 크레딧에 실질적 가치를 부여하기 위한 방안들.

### 12-1. 크레딧 선물하기
- 보유한 크레딧을 다른 유저에게 직접 전달 가능.
- 특히 선배가 신입에게 선물하는 문화를 유도하면, 서비스의 핵심 철학("느슨한 연대", "한국인이라면 서로 끌어준다")이 크레딧 시스템 안에 직접 구현됨.
- 콜드 스타트 완화 효과도 동반: 신입이 가입 시 기본 크레딧(10) 외에 추가로 도움받을 여력이 생김.

### 12-2. 신입 지원 풀 기부
- 개인 간 선물이 아니라, 공용 "크레딧 기부함"에 기부하면 신입들에게 자동으로 분배되는 방식.
- 선물하기보다 간편하지만, 정서적 연결감은 상대적으로 약함.

### 12-3. 제휴 업체 혜택으로 전환
- 부동산·통신사·보험사·어학원 등 워홀 관련 제휴 업체의 할인 쿠폰·혜택으로 크레딧 교환 가능.
- 예: "크레딧 20개 → 유심 구매 할인", "크레딧 30개 → 제휴 보험 상담 무료"
- 11절의 B2B 제휴 수익 모델과 직접 연결되는 방향. 실제 제휴처 확보가 선행되어야 구현 가능.

### 12-4. 우선 매칭/노출권으로 전환
- 크레딧을 소모해 본인의 도움 요청을 게시판 상단에 더 오래 노출시키거나 우선적으로 보이게 함.
- 아직 도움이 더 필요한 유저에게 유용하나, "도움이 더 이상 필요 없는 유저"의 크레딧 소진 고민에는 직접적인 해결책은 아님.

### 12-5. 오프라인 밋업 참가권으로 전환
- 지역 소그룹 오프라인 모임 참가에 크레딧 사용.
- 예: "크레딧 15개 → 이번 달 지역 소그룹 모임 참가권"
- 크레딧이 숫자에서 실제 관계로 전환되는 경험을 제공하며, 11절의 굿즈/오프라인 이벤트 수익 모델과 연결.

### 12-6. 프로필 인증 배지로 전환
- 크레딧을 일정 수준 이상 모으면 "검증된 멤버" 배지 부여.
- 11절의 신원 인증 서비스 수익 모델과 연결되어, 크레딧이 "지금 쓰는 것"이 아니라 "쌓아두면 나중에 더 큰 신뢰가 되는 것"으로 성격이 확장됨.

### 12-7. 코리 캐릭터 꾸미기 아이템 구매
- 크레딧으로 코리 캐릭터의 꾸미기 아이템(의상, 소품 등)을 구매할 수 있게 함.
- 코리는 온보딩 단계부터 홈 화면에 노출되는 서비스의 상징적 캐릭터이므로, 꾸미기 요소는 크레딧 소비처인 동시에 서비스에 대한 애착·재방문 동기를 만드는 역할.
